# Get your robot driving in the Erebus rescue simulator

This site gets you from *"I have a laptop"* to *"my robot is driving around inside a maze on my
screen"* — with a picture for every click, and every confusing word explained the first time it
shows up.

**Who this is for:** you're in grade 9 or up, and you've never set any of this up before. You
don't need to already know how to program. You just need to be able to download a file and find
it again afterwards.

## What you'll build

By the end of this guide, you'll have a real robot — the round one below — driving itself around a
maze inside a program called Webots, using code you can already see and will soon edit yourself.

![The Erebus world1 maze with the e-puck robot, captured from a real run](assets/real/world1-first-run.png)

*The `world1` maze and robot — a real screenshot from running Erebus during the making of this
guide. The grey squares are checkpoints, the coloured tiles are hazards, and the round object near
the top is your robot.*

## The four steps

1. **[Before you start](before-you-start.md)** — the three pieces you'll install (Python, Webots,
   Erebus), and what you need on hand.
2. **Install** — set it up on [Windows](install-windows.md), [macOS](install-mac.md), or
   [Linux](install-linux.md). Pick the page for the computer you're using right now.
3. **[Your first run](first-run.md)** — open the maze world and watch the robot move for the
   first time.
4. **[Make it move](make-it-move.md)** — change one line of the robot's code yourself, and see it
   do something new. This is your first taste of programming the robot.

Stuck at any point? **[When it goes wrong](troubleshooting.md)** is a single page indexed by what
you're seeing on screen. **[Glossary](glossary.md)** explains every unfamiliar word.

## Why this guide exists

The official Erebus install docs are accurate but written for someone who already knows terms
like "PATH" and "controller." This guide adds the missing screenshots, plain-English explanations,
and "you should now see…" checks after every step, so nothing is left to guesswork. See
`UPSTREAM-ISSUES.md` in this project's repository for the specific gaps we found and filled.

Ready? Start with **[Before you start](before-you-start.md)**.
